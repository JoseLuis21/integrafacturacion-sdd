# AI Development Stack

This project is designed for **AI-assisted Specification Driven Development (SDD)**.

To work effectively with the framework, the following AI tools are recommended.

---

# Core AI Tools

## Claude Code

Recommended for:

- implementing modules from specs
- reviewing specs
- refactoring modules
- implementing use cases
- generating tests

Claude is particularly good at:
- reasoning about architecture
- long context understanding
- multi-file refactors

---

## OpenAI Codex / GPT

Recommended for:

- fast boilerplate generation
- implementing endpoints
- generating DTOs and schemas
- generating tests

---

## Cursor (Optional)

Useful for:

- inline AI editing
- quick refactors
- code navigation

---

# Typical AI Usage Flow

1. Write module spec
2. Generate module JSON
3. Generate boilerplate
4. Implement use cases
5. Generate tests
6. Review with AI

---

# Important Rule

AI should **assist implementation**, not replace architecture decisions.

Architecture, security rules, and business logic must always be validated by developers.
